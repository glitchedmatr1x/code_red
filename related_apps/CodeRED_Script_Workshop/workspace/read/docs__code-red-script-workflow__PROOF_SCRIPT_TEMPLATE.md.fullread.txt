# Proof Script Template

Use this before wiring any script command into a real menu.

## Proof name

```text
CodeRED_<feature>_Proof_<YYYY-MM-DD>
```

## Feature

Describe the exact feature in one sentence.

Example:

```text
Read the player position and print/log X/Y/Z without changing game state.
```

## Lane

Choose one:

- Trainer / ScriptHook / RedHook
- SC-CL
- RPF/resource
- Cleanroom research
- Demo/arcade

## Evidence

Fill this before coding:

```text
native/wrapper:
signature:
source path/reference:
evidence type:
known safe ranges:
known failure states:
```

## Tiny proof rules

The proof must:

- do one thing only
- avoid unrelated UI work
- print or log its result
- fail safely
- be easy to remove
- not patch live archives

## Runtime log format

Use a compact log entry:

```json
{
  "feature": "",
  "lane": "",
  "command": "",
  "actor_handle": null,
  "destination": null,
  "result": "",
  "errors": []
}
```

## Pass/fail criteria

Pass:

- exact signature is verified
- proof runs or compiles as expected
- failure path is safe
- result is logged

Fail:

- guessed signature
- unverified handle use
- crash/hang
- no proof log
- requires unrelated full-menu code

## Promotion

Only after passing proof:

```text
research -> verified-native -> runtime-proof or compile-proof -> menu-ready
```
