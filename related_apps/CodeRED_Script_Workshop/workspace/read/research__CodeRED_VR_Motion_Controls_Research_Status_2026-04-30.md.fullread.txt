# Code RED VR Motion / Controls Research Status

Date: 2026-04-30

## Status

This pass established the VR direction as a real Code RED research lane and wrote down the target in the repository. The current goal is not to jump straight into headset rendering first. The correct first milestone is stable first-person body and weapon presentation.

## Goal Recorded

Reference goal recorded:

- `https://github.com/Hotrian/OpenVRDesktopDisplayPortal`

Purpose:

- Treat this as a reference target for a desktop/OpenVR portal-style display path.
- Use it after camera/weapon/body correctness is proven.

## Current Best Plan

The working path is:

1. Camera/first-person tuning.
2. Weapon hand/IK/ACT tuning.
3. NaturalMotion arm/body stability.
4. Input/controller hook bridge.
5. VR display/portal route.
