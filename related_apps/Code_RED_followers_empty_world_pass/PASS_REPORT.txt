Follower / companion status
---------------------------
This pass makes the existing escort, rescue, transport, wagon, chase, and lone-stranger style events more eligible in the wilderness.

The old deny set named "Approach Events / Pathfinding issues" originally blocked:
- event_stickup
- event_lone_stranger
- beat_crime_wagonthief
- event_gnrc_rescue_beacon
- event_hanging

This pass leaves only event_stickup in that deny set, so the less hostile approach/follower-like events can try to run.

Important:
- This does not inject Dutch/Javier/Bill as guaranteed permanent companions.
- This does not invent new character templates.
- It is the safest current step toward follower-like world behavior because it unlocks existing event routes without corrupting downstream scripts or unknown spawn templates.
- True permanent named followers likely needs a script/taskmachine hook or a known companion/freemode template path.

Install / test order
--------------------
1. Back up your original content.rpf.
2. Replace the test copy of content.rpf with the included patched content.rpf.
3. Start the game in a clean test environment.
4. Go to wilderness / empty areas far from roads and major paths.
5. Let time pass and travel through different terrain bands.
6. Watch for:
   - rescue beacons
   - escort/criminal transport style activity
   - wagon thief activity
   - broken wagon events
   - campfire activity
   - criminal chase / animal chase activity
   - landmark attacks

If the game crashes or hangs:
1. Restore the original content.rpf.
2. Report whether the crash happened on boot, region load, or event spawn.
3. Keep the pass report and manifest so we can narrow the next edit.
