Cross-archive notes
- Best direct gameplay patch targets remain content.rpf + tune_d11generic.rpf: placementglobals, factionrelations, AI .tr files, level.pop, default.traffic, refgroups, vehicle/turret/trainmanager/tune files.
- navres.rpf is a validation/safety layer: battleSet mappings, cover/nav resources, and movable nav entries for cars, trucks, Gatling/prison wagons, trains, boats, minecarts.
- gringores.rpf is small but valuable for location-specific shop/saloon/patron gringo context, especially thieveslanding.wgd.
- act.rpf contains many WAT/RSC action resources for lasso, hogtie, robbery, shopkeeping, vehicles, coward/duck behavior, lawman help, and bandito/outlaw actions. Treat as reference until Code RED has a safer WAT decoder/rebuilder.
- naturalmotion.rpf can make combat feel better through Euphoria/NaturalMotion behaviours; it should polish hits/falls/horse reactions, not drive spawning.
- vfx.rpf is likely a future polish target for muzzle smoke, fire, explosion, dust, impact, and train/battle chaos visuals.
- treeres/grassres can support wilderness visual density, but they are not likely to control events directly.
- mapres, psocache, shaders, fonts, textures are mostly render/UI/cache/media resources and are low priority for faction-war logic.
- blackwater.rpf and the DLC rpfs are worth targeted study for region-specific Blackwater/territory content, but broad scan did not make them higher priority than content/tune/gringores/act/navres.

Recommended next technical upgrade for Code RED
Add a Resource Discovery tab that classifies each RPF entry as XML/TR/brace DSL/table/RSC-with-Zstandard/binary/media, then tags it with gameplay terms like event, camp, vehicle, train, lasso, shop, faction, nav, cover, or VFX. This would make these scans repeatable from the UI.
